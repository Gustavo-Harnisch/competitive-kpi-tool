# Project definition

## Candidate names

1. Competitive KPI Tool
2. PracticePulse CP
3. EloRoutine
4. CP Progress Ledger
5. DailySolve

## Selected name

**Competitive KPI Tool** is selected because it is explicit, searchable, technically neutral, and aligned with the existing repository name.

## GitHub description

Local-first desktop KPI tracker for competitive programming practice, built with Lazarus, Object Pascal, and SQLite.

## Scope

- Record practice date, platform, solved problems, minutes, and notes.
- Configure a daily problem goal.
- Calculate daily completion, seven-day average, total sessions, and current streak.
- Store data locally in SQLite.
- Export data to standards-oriented CSV and readable SQL.
- Run on Linux, Windows, and macOS desktop environments after native validation.

## Out of scope for v1.0

- Mobile applications.
- Cloud synchronization or user accounts.
- Automated Codeforces/AtCoder scraping.
- AI coaching or rating prediction.
- Background system daemons and platform-native notification integrations.

## Users and use cases

Primary users are competitive-programming students who want a disciplined local practice log. Secondary users are recruiters and reviewers evaluating database, desktop, quality, and automation skills.

Core use cases: record a session, inspect progress, detect practice gaps, export data for spreadsheet analysis, create a SQL backup, and demonstrate engineering decisions.

## Value proposition

The tool combines daily behavioral feedback with verifiable engineering artifacts. It is useful without an internet connection and keeps the user's data under local control.
